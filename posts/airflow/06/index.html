[Airflow - 6лӢЁкі„: KubernetesExecutor вҖ” мҷң CeleryExecutor лҢҖмӢ  м„ нғқн–ҲлҠ”к°Җ]

Airflow KubernetesExecutor: Task кІ©лҰ¬мҷҖ лҸҷм Ғ лҰ¬мҶҢмҠӨ н• лӢ№#

рҹ§  мӢңлӮҳлҰ¬мҳӨ м„ӨлӘ…#

“AirflowлҘј Kubernetes мң„м—җ мҳ¬лҰ¬л©ҙ Executorк°Җ м„ё к°Җм§Җмһ…лӢҲлӢӨ.

LocalExecutor, CeleryExecutor, KubernetesExecutor.

мқҙ мӨ‘м—җм„ң MLOps нҢҢмқҙн”„лқјмқём—җ к°ҖмһҘ м Ғн•©н•ң кІғмқ„ м„ нғқн•ҳлҠ” кіјм •мқҖ

лӢЁмҲңн•ң м„Өм№ҳ мҳөм…ҳ м„ нғқмқҙ м•„лӢҲлқј мҡҙмҳҒ кө¬мЎ°м—җ лҢҖн•ң кІ°м •мһ…лӢҲлӢӨ.”

“мқҙ н”„лЎңм қнҠём—җм„ңлҠ” CeleryExecutor лҢҖмӢ  KubernetesExecutorлҘј м„ нғқн–ҲмҠөлӢҲлӢӨ.

мқҙ кёҖмқҖ к·ё мқҙмң мҷҖ мӢӨм ң кө¬м„ұ л°©лІ•мқ„ м„ӨлӘ…н•©лӢҲлӢӨ.”

рҹҺҜ н•өмӢ¬ мҡ”м•Ҫ#

  • Executor м„ нғқ кё°мӨҖ: Task кІ©лҰ¬ / лҸҷм Ғ лҰ¬мҶҢмҠӨ / Worker кҙҖлҰ¬ л№„мҡ©
  • KubernetesExecutor м„ нғқ мқҙмң 
    • Taskл§ҲлӢӨ лҸ…лҰҪ Pod вҶ’ нҷҳкІҪ кІ©лҰ¬ ліҙмһҘ
    • Broker(Redis/RabbitMQ) л¶Ҳн•„мҡ” вҶ’ мҡҙмҳҒ ліөмһЎлҸ„ к°җмҶҢ
    • Idle Worker м—ҶмқҢ вҶ’ лҰ¬мҶҢмҠӨ лӮӯл№„ м ңкұ°
    • Task лӢЁмң„ лҰ¬мҶҢмҠӨ м •мқҳ вҶ’ ML н•ҷмҠө/м„ңл№ҷ мһ‘м—…м—җ м Ғн•©
  • мӢӨм ң кө¬м„ұ мҡ”мҶҢ
    • ServiceAccount + RBAC (Pod мғқм„ұ к¶Ңн•ң)
    • pod_template_file (Task Pod н‘ңмӨҖ м •мқҳ)
    • KubernetesPodOperator (м»ӨмҠӨн…Җ Task Pod)
  • мҡҙмҳҒ мЈјмқҳмӮ¬н•ӯ
    • Cold start: Pod мҠӨмјҖмӨ„л§Ғ м§Җм—° (1030s)
    • Image pull policy: IfNotPresent кё°ліё к¶ҢмһҘ
    • лЎңк·ё м§ҖмҶҚм„ұ: NFS лҳҗлҠ” Remote Logging н•„мҲҳ

2пёҸвғЈ KubernetesExecutorлҘј м„ нғқн•ң мқҙмң #

(1) Task кІ©лҰ¬#

CeleryExecutorмқҳ WorkerлҠ” н”„лЎңм„ёмҠӨ лӢЁмң„лЎң кІ©лҰ¬лҗ©лӢҲлӢӨ.

мҰү, к°ҷмқҖ Worker Podм—җм„ң лӢӨлҘё Taskл“Өмқҙ лҸҷмқјн•ң Python нҷҳкІҪмқ„ кіөмң н•©лӢҲлӢӨ.

CeleryExecutor
Worker Pod вҶ’ Task A, Task B, Task C (к°ҷмқҖ нҷҳкІҪ кіөмң )

KubernetesExecutorлҠ” Taskл§ҲлӢӨ Podк°Җ л¶„лҰ¬лҗ©лӢҲлӢӨ.

KubernetesExecutor
Task A вҶ’ Pod-A (лҸ…лҰҪ мӢӨн–ү, мҷ„лЈҢ нӣ„ мӮӯм ң)
Task B вҶ’ Pod-B (лҸ…лҰҪ мӢӨн–ү, мҷ„лЈҢ нӣ„ мӮӯм ң)
Task C вҶ’ Pod-C (лҸ…лҰҪ мӢӨн–ү, мҷ„лЈҢ нӣ„ мӮӯм ң)

ML нҢҢмқҙн”„лқјмқём—җм„ңлҠ” Taskл§ҲлӢӨ нҢЁнӮӨм§Җ мқҳмЎҙм„ұмқҙ лӢӨлҘј мҲҳ мһҲмҠөлӢҲлӢӨ.

мҳҲлҘј л“Өм–ҙ feature мғқм„ұ TaskмҷҖ model н•ҷмҠө TaskлҠ” м„ңлЎң лӢӨлҘё лқјмқҙлёҢлҹ¬лҰ¬к°Җ н•„мҡ”н•  мҲҳ мһҲмҠөлӢҲлӢӨ.

KubernetesExecutorлҠ” мқҙ л¬ём ңлҘј кө¬мЎ°м ҒмңјлЎң н•ҙкІ°н•©лӢҲлӢӨ.

(2) Broker л¶Ҳн•„мҡ”#

CeleryExecutorлҠ” л°ҳл“ңмӢң Brokerк°Җ н•„мҡ”н•©лӢҲлӢӨ.

CeleryExecutor кө¬мЎ°
Scheduler вҶ’ Redis/RabbitMQ(Broker) вҶ’ Worker

BrokerлҠ” лі„лҸ„ м„ңл№„мҠӨмқҙлҜҖлЎң лӢӨмқҢмқҙ м¶”к°Җлҗ©лӢҲлӢӨ.

  • Broker м„Өм№ҳ л°Ҹ мҡҙмҳҒ
  • Broker мһҘм•  лҢҖмқ‘
  • Broker лӘЁлӢҲн„°л§Ғ

KubernetesExecutorлҠ” Broker м—Ҷмқҙ Kubernetes APIлҘј м§Ғм ‘ мӮ¬мҡ©н•©лӢҲлӢӨ.

KubernetesExecutor кө¬мЎ°
Scheduler вҶ’ Kubernetes API вҶ’ Task Pod

мҡҙмҳҒ м»ҙнҸ¬л„ҢнҠёк°Җ мӨ„м–ҙл“ лӢӨлҠ” кІғмқҖ мһҘм•  л°ңмғқ м§Җм җмқҙ мӨ„м–ҙл“ңлҠ” кІғкіј к°ҷмҠөлӢҲлӢӨ.

(3) Idle Worker м—ҶмқҢ#

CeleryExecutorлҠ” Workerк°Җ н•ӯмғҒ мӢӨн–ү мӨ‘мһ…лӢҲлӢӨ.

DAGк°Җ мӢӨн–үлҗҳм§Җ м•ҠлҠ” мӢңк°„м—җлҸ„ Worker PodлҠ” лҰ¬мҶҢмҠӨлҘј м җмң н•©лӢҲлӢӨ.

KubernetesExecutorлҠ” Taskк°Җ мӢӨн–үлҗ  л•Ңл§Ң Podк°Җ мғқм„ұлҗ©лӢҲлӢӨ.

мқҙ кө¬мЎ°лҠ” лҰ¬мҶҢмҠӨ л№„мҡ© мёЎл©ҙм—җм„ң мң лҰ¬н•©лӢҲлӢӨ.

(4) Task лӢЁмң„ лҰ¬мҶҢмҠӨ м •мқҳ#

ML TaskлҠ” лҰ¬мҶҢмҠӨ мҡ”кө¬мӮ¬н•ӯмқҙ м ңк°Ғк°Ғмһ…лӢҲлӢӨ.

feature мғқм„ұ Task: CPU лӮ®мқҢ, Memory ліҙнҶө
model н•ҷмҠө Task:   CPU лҶ’мқҢ, Memory л§ҺмқҢ
drift кІҖмӮ¬ Task:   CPU лӮ®мқҢ, Memory лӮ®мқҢ

KubernetesExecutorлҠ” Taskл§ҲлӢӨ лҰ¬мҶҢмҠӨлҘј лӢӨлҘҙкІҢ м •мқҳн•  мҲҳ мһҲмҠөлӢҲлӢӨ.

3пёҸвғЈ мӢӨм ң RBAC + ServiceAccount кө¬м„ұ#

KubernetesExecutorк°Җ Task PodлҘј мғқм„ұн•ҳл Өл©ҙ Kubernetes RBAC к¶Ңн•ңмқҙ н•„мҡ”н•©лӢҲлӢӨ.

(1) ServiceAccount#

# envs/dev/airflow/serviceaccount.yaml
apiVersion: v1
kind: ServiceAccount
metadata:
  name: airflow-dev
  namespace: airflow-dev

(2) Role (Pod мғқм„ұ/мӮӯм ң/мЎ°нҡҢ к¶Ңн•ң)#

# envs/dev/airflow/role.yaml
apiVersion: rbac.authorization.k8s.io/v1
kind: Role
metadata:
  name: airflow-dev-role
  namespace: airflow-dev
rules:
  - apiGroups: [""]
    resources: ["pods"]
    verbs: ["get", "list", "watch", "create", "delete", "patch"]
  - apiGroups: [""]
    resources: ["pods/log"]
    verbs: ["get", "list"]
  - apiGroups: [""]
    resources: ["pods/exec"]
    verbs: ["create"]
  - apiGroups: [""]
    resources: ["events"]
    verbs: ["get", "list"]
  - apiGroups: [""]
    resources: ["configmaps"]
    verbs: ["get", "list"]

(3) RoleBinding#

# envs/dev/airflow/rolebinding.yaml
apiVersion: rbac.authorization.k8s.io/v1
kind: RoleBinding
metadata:
  name: airflow-dev-rolebinding
  namespace: airflow-dev
roleRef:
  apiGroup: rbac.authorization.k8s.io
  kind: Role
  name: airflow-dev-role
subjects:
  - kind: ServiceAccount
    name: airflow-dev
    namespace: airflow-dev

Airflow Schedulerк°Җ мӢӨн–ү мӨ‘мқё NamespaceмҷҖ Task Podк°Җ мғқм„ұлҗҳлҠ” Namespaceк°Җ к°ҷлӢӨл©ҙ Roleмқҙ м¶©л¶„н•©лӢҲлӢӨ.

лӢӨлҘё Namespaceм—җ Task PodлҘј мғқм„ұн•ҙм•ј н•ңлӢӨл©ҙ ClusterRoleкіј ClusterRoleBindingмқҙ н•„мҡ”н•©лӢҲлӢӨ.

4пёҸвғЈ pod_template_file: Task Pod н‘ңмӨҖ м •мқҳ#

KubernetesExecutorлҠ” Task Podмқҳ кё°ліё н…ңн”ҢлҰҝмқ„ pod_template_fileлЎң м •мқҳн•©лӢҲлӢӨ.

мқҙ нҢҢмқјмқҙ м—Ҷмңјл©ҙ кё°ліёк°’мңјлЎң лҸҷмһ‘н•ҳм§Җл§Ң,

мҡҙмҳҒ нҷҳкІҪм—җм„ңлҠ” л°ҳл“ңмӢң лӘ…мӢңм ҒмңјлЎң м •мқҳн•ҳлҠ” кІғмқҙ мўӢмҠөлӢҲлӢӨ.

# pod_template.yaml

metadata:
  name: base-worker

мқҙ н…ңн”ҢлҰҝмқҖ лӢӨмқҢмқ„ н‘ңмӨҖнҷ”н•©лӢҲлӢӨ.

  • ServiceAccount: RBAC к¶Ңн•ң мһҗлҸҷ л¶Җм—¬
  • ImagePullPolicy: л¶Ҳн•„мҡ”н•ң мқҙлҜём§Җ мһ¬лӢӨмҡҙлЎңл“ң л°©м§Җ
  • Resources: кё°ліё лҰ¬мҶҢмҠӨ ліҙмһҘ
  • Logs PVC: Task лЎңк·ё NFS мҳҒкө¬ м ҖмһҘ

# charts/airflow/values-dev.yaml

  kubernetes:
    pod_template_file: /opt/airflow/pod_template.yaml

5пёҸвғЈ KubernetesPodOperator (KPO) мӢӨм „ мҳҲмӢң#

Taskл§ҲлӢӨ лӢӨлҘё мқҙлҜём§Җ, лӢӨлҘё лҰ¬мҶҢмҠӨлҘј мӮ¬мҡ©н•ҙм•ј н•  л•Ң KPOлҘј нҷңмҡ©н•©лӢҲлӢӨ.

from airflow import DAG
from airflow.providers.cncf.kubernetes.operators.pod import KubernetesPodOperator
from kubernetes.client import models as k8s
from datetime import datetime

with DAG(
    dag_id="kpo_example_dev",
    start_date=datetime(2026, 1, 1),
    schedule=None,
    catchup=False,
    tags=["kpo", "dev"],
) as dag:

    feature_task = KubernetesPodOperator(
        task_id="build_features",
        name="feature-builder",
        namespace="airflow-dev",
        image="my-feature-image:1.0.0",
        image_pull_policy="IfNotPresent",
        service_account_name="airflow-dev",
        resources=k8s.V1ResourceRequirements(
            requests={"cpu": "500m", "memory": "512Mi"},
            limits={"cpu": "1", "memory": "1Gi"},
        ),
        env_vars={
            "S3_BUCKET": "my-feature-bucket",
            "ENV": "dev",
        },
        is_delete_operator_pod=True,      # мҷ„лЈҢ нӣ„ Pod мӮӯм ң
        get_logs=True,                    # Task лЎңк·ё Airflow UIм—җ н‘ңмӢң
        do_xcom_push=False,
    )

    train_task = KubernetesPodOperator(
        task_id="train_model",
        name="model-trainer",
        namespace="airflow-dev",
        image="my-train-image:1.0.0",
        image_pull_policy="IfNotPresent",
        service_account_name="airflow-dev",
        resources=k8s.V1ResourceRequirements(
            # н•ҷмҠө TaskлҠ” лҰ¬мҶҢмҠӨлҘј лҚ” л§Һмқҙ н• лӢ№
            requests={"cpu": "2", "memory": "4Gi"},
            limits={"cpu": "4", "memory": "8Gi"},
        ),
        env_vars={
            "MLFLOW_TRACKING_URI": "http://mlflow-dev-service.mlflow-dev.svc.cluster.local:5000",
        },
        is_delete_operator_pod=True,
        get_logs=True,
    )

    feature_task >> train_task

is_delete_operator_pod=TrueлҠ” Task мҷ„лЈҢ нӣ„ PodлҘј м •лҰ¬н•©лӢҲлӢӨ.

мҡҙмҳҒ нҷҳкІҪм—җм„ңлҠ” л°ҳл“ңмӢң м„Өм •н•ҙм„ң Pod лҲ„м Ғмқ„ л°©м§Җн•ҙм•ј н•©лӢҲлӢӨ.

6пёҸвғЈ мҡҙмҳҒ мЈјмқҳмӮ¬н•ӯ#

(1) Cold Start м§Җм—°#

Task нҠёлҰ¬кұ° вҶ’ Pod мғқм„ұ мҡ”мІӯ вҶ’ мҠӨмјҖмӨ„л§Ғ вҶ’ мқҙлҜём§Җ Pull вҶ’ м»Ён…Ңмқҙл„Ҳ мӢңмһ‘ вҶ’ Task мӢӨн–ү

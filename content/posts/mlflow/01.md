## MLflow Model Registry: Stage 방식의 한계와 alias 기반 전환

---

## 🧠 시나리오 설명

> "MLflow 2.x부터 Staging/Production Stage가 deprecated됩니다.
>
>
> 대신 Registered Model Alias 방식이 공식 권장 패턴이 되었습니다."
>
> "이 변화는 단순한 API 교체가 아닙니다.
>
> 모델 버전 관리의 책임 구조가 바뀌는 것입니다."
>
> "이 글은 Stage 방식의 한계를 분석하고,
>
> alias 기반 Champion/Challenger 패턴과
>
> FastAPI와 연동한 zero-downtime hot swap 구조를 설명합니다."

---

### 🎯 핵심 요약

- **Stage 방식 한계**: 고정된 Stage 이름(Staging/Production)은 운영 유연성 부족
- **alias 방식 전환**: `@champion`, `@challenger` 등 의미 있는 이름으로 버전 지정
- **Champion/Challenger 패턴**: 운영 모델(A)과 검증 모델(B)을 동시에 서빙
- **hot swap 구조**: FastAPI `/variant/{alias}/reload` → MLflow alias 조회 → 모델 교체
- **zero-downtime**: 모델 교체 시 서비스 중단 없음 (메모리 내 교체)
- **MLflow FQDN**
    - dev: `http://mlflow-dev-service.mlflow-dev.svc.cluster.local:5000`
    - prod: `http://mlflow-prod-service.mlflow-prod.svc.cluster.local:5000`

## 1️⃣ Stage 방식의 한계

MLflow 1.x 시대에는 모델 버전에 Stage를 부여했습니다.

```python
# 구방식 (deprecated)
client.transition_model_version_stage(
    name="best_model",
    version=3,
    stage="Production",  # None / Staging / Production / Archived
)
```

이 방식에는 다음 문제가 있습니다.

**고정된 Stage 이름**

Stage는 `None`, `Staging`, `Production`, `Archived` 네 가지만 존재합니다.

운영에서 필요한 `champion`, `challenger`, `shadow`, `rollback` 같은 개념을 표현할 수 없습니다.

**동시 서빙 불편**

`Production` Stage에는 하나의 버전만 있는 것이 일반적입니다.

Champion/Challenger 패턴에서 두 모델을 동시에 서빙하려면 Stage 이름만으로는 구분이 어렵습니다.

**MLflow 2.x 공식 입장**

```
MLflow 2.x에서 Model Version Stage는 deprecated됩니다.
대신 Registered Model Alias를 사용하세요.
```

## 2️⃣ alias 방식으로 전환

alias는 특정 모델 버전에 이름을 부여하는 방식입니다.

```python
from mlflow.tracking import MlflowClient

client = MlflowClient()

# alias 설정: version 3에 champion alias 부여
client.set_registered_model_alias(
    name="best_model",
    alias="champion",
    version=3,
)

# alias 설정: version 4에 challenger alias 부여
client.set_registered_model_alias(
    name="best_model",
    alias="challenger",
    version=4,
)
```

alias로 모델을 로드할 때는 다음 URI를 사용합니다.

```python
import mlflow.pyfunc

# champion 모델 로드
model = mlflow.pyfunc.load_model("models:/best_model@champion")

# challenger 모델 로드
model = mlflow.pyfunc.load_model("models:/best_model@challenger")
```

alias의 장점은 다음과 같습니다.

```
models:/best_model@champion     → 현재 운영 모델
models:/best_model@challenger   → 검증 중인 모델
models:/best_model@shadow       → Shadow 테스트 모델
models:/best_model@rollback     → 롤백 대상 모델
```

이름이 명시적이므로 운영 의도가 코드에 직접 드러납니다.

## 3️⃣ Champion/Challenger 패턴

Champion은 현재 운영 중인 모델, Challenger는 검증 중인 새 모델입니다.

```
트래픽 분배 예시 (A/B 테스트)
요청 → FastAPI
          ↓
   client_id 해시 기반 분기
          ↓
   90% → champion (alias=A)
   10% → challenger (alias=B)
```

이 구조의 핵심은 다음입니다.

- Champion을 교체하지 않고 Challenger와 동시에 운영
- Challenger 성능이 충분하면 alias를 재지정해서 Champion으로 승격
- 서비스 중단 없이 모델 교체

**alias 재지정으로 승격**

```python
# Challenger(version 4)를 Champion으로 승격
client.set_registered_model_alias(
    name="best_model",
    alias="champion",  # A alias를 version 4로 재지정
    version=4,
)

# 기존 Champion(version 3)은 rollback alias로 유지
client.set_registered_model_alias(
    name="best_model",
    alias="rollback",
    version=3,
)
```

alias만 재지정하면 됩니다. 모델 파일은 그대로입니다.

## 4️⃣ FastAPI와의 hot swap 구조

### 전체 흐름

```
[alias 재지정]
MLflow Registry: best_model@champion → version 4

      ↓

[hot swap 요청]
POST /variant/A/reload
  Header: x-token: <RELOAD_SECRET_TOKEN>

      ↓

[FastAPI 처리]
1. x-token 검증
2. MLflow에서 alias=A 조회
   → model_uri: models:/best_model@A
   → version: 4, run_id: abc123...
3. mlflow.pyfunc.load_model(model_uri)
4. app.state.models["A"] 교체
5. Slack 알림 전송

      ↓

[결과]
서비스 중단 없이 version 4 모델로 교체 완료
```

### model_loader.py

```python
# services/model_loader.py
import mlflow.pyfunc
from mlflow.tracking import MlflowClient
from utils.slack_alerts import send_slack_alert
from loguru import logger
from core.config import settings


def load_model_by_alias(alias: str):
    """
    MLflow Registry에서 alias로 모델을 로드합니다.

    alias 예시: "A" → models:/best_model@A
    실제 운영에서 alias는 "A", "B" 또는 "champion", "challenger" 등
    """
    try:
        mlflow.set_tracking_uri(settings.mlflow_tracking_uri)
        client = MlflowClient()

        model_uri = f"models:/{settings.model_name}@{alias}"
        model = mlflow.pyfunc.load_model(model_uri)

        # alias가 가리키는 실제 버전 정보 조회
        version_info = client.get_model_version_by_alias(settings.model_name, alias)

        logger.info(
            f"✅ 모델 로딩 성공: alias={alias}, "
            f"version={version_info.version}, "
            f"run_id={version_info.run_id}"
        )
        return {
            "model": model,
            "info": {
                "model_name": settings.model_name,
                "alias": alias,
                "version": version_info.version,
                "run_id": version_info.run_id,
                "model_uri": model_uri,
            },
        }
    except Exception as e:
        logger.error(f"❌ 모델 로딩 실패: alias={alias}, error={e}")
        send_slack_alert(f"❌ 모델 로딩 실패: alias={alias}, {e}")
        return None
```

### reload.py

```python
# routes/reload.py
from fastapi import APIRouter, Header, HTTPException, Request
from services.model_loader import load_model_by_alias
from utils.logger import logger
from utils.slack_alerts import send_slack_alert
from core.config import settings

router = APIRouter()


@router.post("/variant/{alias}/reload")
def reload_model(
    alias: str,
    request: Request,
    x_token: str | None = Header(default=None),
):
    # 1. 토큰 검증
    if x_token != settings.reload_secret_token:
        raise HTTPException(status_code=403, detail="Invalid token")

    # 2. 새 모델 로드
    new_entry = load_model_by_alias(alias)
    if new_entry is None:
        raise HTTPException(
            status_code=500,
            detail=f"모델 로딩 실패: alias={alias}"
        )

    # 3. 메모리 내 교체 (zero-downtime)
    if not hasattr(request.app.state, "models"):
        request.app.state.models = {}
    request.app.state.models[alias] = new_entry

    info = new_entry["info"]
    logger.info(
        f"🔄 hot swap 완료: alias={alias}, "
        f"version={info['version']}, run_id={info['run_id']}"
    )
    send_slack_alert(
        f"🔄 모델 교체 완료\n"
        f"alias={alias}, version={info['version']}, run_id={info['run_id']}"
    )

    return {
        "status": "reloaded",
        "alias": alias,
        "version": info["version"],
        "run_id": info["run_id"],
        "model_uri": info["model_uri"],
    }
```

## 5️⃣ alias 정보 조회: mlflow_meta.py

현재 어떤 버전이 어떤 alias를 가지고 있는지 확인하는 유틸리티입니다.

```python
# utils/mlflow_meta.py
import mlflow
from mlflow.tracking import MlflowClient


def get_alias_info(model_name: str, alias: str, tracking_uri: str) -> dict:
    """
    특정 alias의 모델 버전 정보를 반환합니다.

    반환값 예시:
    {
        "model_name": "best_model",
        "alias": "champion",
        "version": "3",
        "run_id": "abc123...",
        "model_uri": "models:/best_model@champion",
        "status": "READY",
        "tags": {...}
    }
    """
    mlflow.set_tracking_uri(tracking_uri)
    client = MlflowClient()

    version_info = client.get_model_version_by_alias(model_name, alias)

    return {
        "model_name": model_name,
        "alias": alias,
        "version": version_info.version,
        "run_id": version_info.run_id,
        "model_uri": f"models:/{model_name}@{alias}",
        "status": version_info.status,
        "tags": version_info.tags,
    }


def list_aliases(model_name: str, tracking_uri: str) -> list[dict]:
    """
    등록된 모델의 모든 alias 목록을 반환합니다.
    """
    mlflow.set_tracking_uri(tracking_uri)
    client = MlflowClient()

    model = client.get_registered_model(model_name)
    aliases = []
    for alias_info in model.aliases:
        aliases.append({
            "alias": alias_info,
            "version": model.aliases[alias_info],
        })
    return aliases
```

## 6️⃣ Airflow DAG에서 alias 자동 관리

E2E 파이프라인에서 모델 학습 후 alias를 자동으로 관리합니다.

```python
# dags/mlops_lib/core/registry.py (핵심 부분)
import mlflow
from mlflow.tracking import MlflowClient


def promote_to_champion(model_name: str, version: str, tracking_uri: str):
    """
    기존 champion을 rollback으로 내리고
    새 버전을 champion으로 승격합니다.
    """
    mlflow.set_tracking_uri(tracking_uri)
    client = MlflowClient()

    # 기존 champion 버전 확인
    try:
        old_champion = client.get_model_version_by_alias(model_name, "champion")
        # 기존 champion을 rollback alias로 보존
        client.set_registered_model_alias(
            name=model_name,
            alias="rollback",
            version=old_champion.version,
        )
    except Exception:
        pass  # 기존 champion이 없으면 그냥 진행

    # 새 버전을 champion으로 승격
    client.set_registered_model_alias(
        name=model_name,
        alias="champion",
        version=version,
    )


def assign_as_challenger(model_name: str, version: str, tracking_uri: str):
    """
    새로 학습된 버전을 challenger alias로 등록합니다.
    """
    mlflow.set_tracking_uri(tracking_uri)
    client = MlflowClient()

    client.set_registered_model_alias(
        name=model_name,
        alias="challenger",
        version=version,
    )
```

DAG에서 Promotion/Shadow 분기 후 호출합니다.

```python
# 성능 기준 통과 → champion 승격
if should_promote:
    promote_to_champion(model_name, version, tracking_uri)
    # FastAPI hot swap 요청
    requests.post(
        f"{fastapi_url}/variant/A/reload",
        headers={"x-token": reload_token},
    )
else:
    # Shadow 경로 → challenger로만 등록
    assign_as_challenger(model_name, version, tracking_uri)
```

## 🧩 트러블슈팅

### (1) `MlflowException: Registered model alias ... not found`

alias가 설정되지 않은 상태에서 로드를 시도한 경우입니다.

```python
# alias 목록 확인
client = MlflowClient()
model = client.get_registered_model("best_model")
print(model.aliases)
```

alias가 없으면 먼저 `set_registered_model_alias`로 등록합니다.

### (2) hot swap 후에도 이전 버전이 계속 사용됨

`app.state.models` 교체는 정상이지만 예측에서 계속 이전 모델을 쓰는 경우입니다.

```python
# predict.py에서 모델 참조 방식 확인
# ✅ 정상: request.app.state.models에서 매 요청마다 조회
model_entry = models.get(alias)

# ❌ 문제: 모듈 레벨 전역 변수에 모델을 캐싱하는 경우
# 전역 변수는 reload 시 교체가 반영되지 않음
```

### (3) MLflow Tracking Server 연결 실패

K8s 내부에서 FQDN을 정확히 써야 합니다.

## 🔧 MLOps 실전 연결

alias 기반 모델 관리는 E2E 파이프라인의 핵심 연결 고리입니다.

```
Airflow DAG
  → 모델 학습 + MLflow 등록
  → alias 설정 (champion/challenger/rollback)
  → FastAPI hot swap 요청 (/variant/{alias}/reload)
  → zero-downtime 모델 교체
  → Post-deploy Observability
  → 문제 감지 시 rollback alias로 revert
```

이 구조의 핵심은

**모델 파일을 교체하지 않고 alias 포인터만 바꾸는 것**입니다.

`models:/best_model@champion` URI는 항상 동일하지만,

champion이 가리키는 실제 버전 번호는 alias 재지정으로 언제든 바꿀 수 있습니다.

## 🏁 정리

| 항목 | Stage 방식 (deprecated) | alias 방식 (권장) |
| --- | --- | --- |
| 버전 지정 방식 | 고정 Stage 이름 | 자유로운 alias 이름 |
| 동시 서빙 | 불편함 | champion/challenger 명확 |
| 승격 방법 | Stage 전환 API | alias 재지정 |
| 운영 의도 표현 | 제한적 | champion/challenger/rollback 등 명시 |
| MLflow 2.x 지원 | deprecated | 공식 권장 |

> alias는 단순한 이름표가 아닙니다.
>
> 운영 환경에서 어떤 모델이 어떤 역할을 하는지를
>
> **코드와 URI에 직접 표현하는 수단**입니다.
